# PRD — 인스타그램 포스팅 자동화 도구

> 작성일: 2026-06-19 · 상태: Draft v0.1 · 레퍼런스: YouTube `gz3q1iV6mAE` (분석: [ANALYSIS.md](./ANALYSIS.md))

## 1. 배경 & 문제정의
SNS 운영자/소상공인/마케터는 인스타그램에 **반복적으로 콘텐츠를 발행**해야 하지만 수작업은 시간 소모가 크다. 레퍼런스 영상의 도구는 이 문제를 "다중 계정 + 브라우저 자동화 + 콘텐츠/이미지 자동생성"으로 푼다.

본 제품은 동일한 **생산성 가치(콘텐츠 생성 자동화 + 예약 발행 + 이미지 처리)** 를 제공하되, **약관을 준수하는 공식 경로**로 구현한다.

## 2. 목표 (Goals)
- G1. 본인이 소유/관리 권한이 있는 인스타 비즈니스·크리에이터 계정에 **예약·자동 발행**.
- G2. **콘텐츠 자동 생성**: 템플릿/키워드/해시태그 조합, 캡션 변형(스핀택스/AI).
- G3. **이미지 처리 파이프라인**: 폴더 일괄 처리, 텍스트 오버레이, 필터, 리사이즈/크롭.
- G4. 피드/릴스/이미지·동영상 발행을 **큐 + 스케줄러**로 관리.
- G5. 발행 이력/성공·실패 로그/간단 인사이트 대시보드.

## 3. 비목표 (Non-Goals / Out of Scope) ⚠️
약관·법적 리스크가 큰 레퍼런스 기능은 **명시적으로 제외**한다.
- N1. **대량/타인 계정 팜 운영** (계정당 id·pw 무제한 등록).
- N2. 탐지 회피용 랜덤 딜레이 스푸핑.
- N3. **비공식 브라우저 자동화(셀레니움)로 instagram.com 직접 조작** — 1차 경로에서 배제.
- N4. **스팸/광고 무차별 발행**(텔레그램 영업글 등).
- N5. 좋아요/팔로우/DM 매크로 등 인게이지먼트 봇.

## 4. 컴플라이언스 원칙
- **공식 Instagram Graph API – Content Publishing** 사용 (Business/Creator + 연결된 Facebook Page).
- API 레이트리밋 준수: 계정당 **24시간 50회** 발행 제한 등 공식 한도 내 동작.
- 자격증명은 OAuth 토큰 기반, 평문 비밀번호 저장 금지.
- 사용자에게 약관/리스크 고지, 본인 소유 계정만 연결.

## 5. 타깃 사용자
| 페르소나 | 니즈 |
|---|---|
| 소상공인/브랜드 운영자 | 일관된 피드 운영, 예약 발행, 손쉬운 이미지 편집 |
| 콘텐츠 마케터 | 다계정(본인 관리) 일괄 캠페인, 캡션 변형, 성과 추적 |
| 1인 크리에이터 | 정해진 시간 자동 발행, 해시태그 자동화 |

## 6. 핵심 기능 (Functional Requirements)

### FR-1 계정 연결 & 관리
- FR-1.1 Facebook/Instagram **OAuth 로그인**으로 비즈니스/크리에이터 계정 연결.
- FR-1.2 연결된 계정 목록·토큰 만료 상태 표시, 토큰 자동 갱신.
- FR-1.3 계정별 발행 한도(잔여 횟수) 표시.

### FR-2 콘텐츠 작성 & 자동 생성
- FR-2.1 캡션 에디터: 본문 + 해시태그 + 멘션 + 이모지.
- FR-2.2 **템플릿/변수 조합**: `{제목}`, `{키워드}`, `{해시태그셋}` 치환 + 스핀택스(`{A|B|C}`) 변형 → 레퍼런스의 "제목+키워드+특수문자 조합" 정당화 버전.
- FR-2.3 **해시태그 세트** 저장·랜덤/순서 선택.
- FR-2.4 (옵션) Claude(`claude-opus-4-8`) 기반 캡션 생성/리라이트.
- FR-2.5 캡션 리스트(여러 변형) 순서/랜덤 적용.

### FR-3 이미지/동영상 처리 (레퍼런스 이미지설정 탭 정당화 이식)
- FR-3.1 단일/폴더 일괄 로드, 순서·랜덤 선택.
- FR-3.2 **텍스트 오버레이**: 글자 삽입(2개 슬롯), 폰트/크기, 테두리, 그림자.
- FR-3.3 **필터**: 흑백/세피아 등 프리셋.
- FR-3.4 테두리, **리사이즈/크롭**(1080/1080·4:5·1.91:1 등 IG 규격 프리셋 + 원본).
- FR-3.5 동영상(릴스) 업로드 지원, 썸네일/커버 지정.
- FR-3.6 처리 결과 **미리보기**.

### FR-4 스케줄링 & 발행 큐
- FR-4.1 즉시 발행 / 예약 발행(날짜·시간, 타임존).
- FR-4.2 **발행 큐**: 항목별 상태(대기·발행중·완료·실패) + Progress.
- FR-4.3 계정별 **최소 간격(쿨다운)** 설정.
- FR-4.4 실패 시 재시도 정책, 실패 사유 로깅.
- FR-4.5 미디어 타입: 단일 이미지 / 캐러셀 / 릴스.

### FR-5 이력 & 인사이트
- FR-5.1 발행 이력(성공/실패, 게시물 링크).
- FR-5.2 기본 인사이트(좋아요/도달/노출) — Graph API Insights 범위.
- FR-5.3 CSV 내보내기.

### FR-6 설정/프로젝트
- FR-6.1 설정 저장/불러오기(레퍼런스의 세팅 저장/초기화 대응).
- FR-6.2 프로젝트(캠페인) 단위로 콘텐츠·이미지·일정 묶음 관리.

## 7. 기술 아키텍처

### 7.1 발행 경로 — 결정 필요
| 옵션 | 방식 | 장점 | 단점 |
|---|---|---|---|
| **A. 공식 Graph API (권장)** | IG Content Publishing API | 약관 준수, 안정적, 예약·릴스·캐러셀 지원 | 비즈/크리에이터 계정 + FB 연동 필수, 앱 검수 필요 |
| B. 비공식 라이브러리 (`instagrapi` 등) | 모바일 프라이빗 API | 개인계정도 가능, 검수 불필요 | **약관 위반·차단 리스크**, 비권장 |
| C. 브라우저 자동화 (Playwright) | 웹 UI 조작 | API 미지원 기능 가능 | 깨지기 쉬움·탐지·약관 위반 |

→ **기본은 A(공식 API)**. B/C는 "본인 책임" 실험 모듈로만 분리(기본 비활성).

### 7.2 후보 스택
- **데스크톱 앱**(레퍼런스와 동일 폼팩터): Electron + React + TypeScript, 로컬 SQLite, 이미지 처리 `sharp`.
- 또는 **웹 앱**: Next.js(App Router) on Vercel + Vercel Cron(예약 발행) + Postgres(Marketplace) + Vercel Blob(미디어).
- 이미지 처리: `sharp`(리사이즈/필터/오버레이) 또는 `canvas`.
- 스케줄러: 데스크톱=로컬 큐/cron, 웹=Vercel Cron + 작업 테이블.

### 7.3 데이터 모델(초안)
- `accounts`(id, ig_user_id, name, token_enc, token_expiry, daily_limit, status)
- `media_assets`(id, path/blob_url, type, processed_meta)
- `posts`(id, account_id, caption, asset_ids, type[image|carousel|reel], status, scheduled_at, published_at, ig_media_id, error)
- `templates`(id, name, body, hashtag_sets, spintax)
- `logs`(id, post_id, event, payload, ts)

## 8. UI/UX (화면 구성)
- 좌측 내비: 대시보드 · 계정 · 콘텐츠 작성 · 미디어 · 일정/큐 · 이력 · 설정.
- 콘텐츠 작성: 캡션 에디터 + 템플릿/해시태그 + 미디어 처리 패널 + 미리보기(IG 목업).
- 일정/큐: 캘린더 + 큐 테이블(상태·Progress) — 레퍼런스 테이블 UX 계승.

## 9. 비기능 요구사항
- 보안: 토큰 암호화 저장(OS 키체인/`safeStorage` 또는 KMS), 비밀번호 미저장.
- 안정성: 발행 실패 재시도·멱등성(중복 발행 방지 키).
- 관측성: 구조화 로깅, 발행 성공률 지표.
- i18n: 한국어 우선.
